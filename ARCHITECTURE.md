This project uses a webhook-driven telephony loop: a Python CLI places outbound calls through Twilio, and FastAPI endpoints respond with TwiML for each turn. The caller simulator acts as a patient with scenario-specific objectives (scheduling, cancellation, refill, billing, language support, urgent symptoms, and edge cases), while maintaining coherent turn-taking through greeting detection, interruption handling, and bounded response generation. The conversation engine uses OpenAI for patient utterances with low-temperature settings and concise style constraints to prioritize lucid voice interactions, which is the top evaluation criterion.

Each call is persisted by Call SID under data/calls with session state, turn history, recording metadata, downloaded mp3, and a canonical transcript. The post-call pipeline is resilient: reconcile-calls backfills status and recordings from Twilio if callbacks were missed, materialize-transcripts produces canonical transcript artifacts (audio-derived when available, turn-log fallback), and generate-report produces structured bug findings for analysis. Key design choices are reliability and evidence quality over complexity: strict destination guardrails, paced batch dialing, explicit call-duration limits, and reproducible artifact generation to satisfy the assignment requirement of 10+ full calls with both recordings and transcripts.
